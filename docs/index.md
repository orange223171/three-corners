# Три угла
Три угла — минималистичная стратегия на поле из треугольников. [Правила игры](game_rules/game_rules.md).

# Модули
## [core](core/core.md)
- Предоставляет определения основных констант, структур и функций, используемых клиентом и сервером;
## [3c](server/3c/3c.md)
- Предоставляет структуру для хранения текущего состояния игры;
- Предоставляет функции управления процессом игры;
- Выполняет проверку действий игроков.
## [network](network/network.md)
- Содержит три подмодуля: network-core, network-client, network-server;
- Предоставляет функционал для обмена данными между клиентом и сервером.
## [bd](server/bd/bd.md)
- Содержит функции для выполнения запросов к базе данных;
- Возвращает информацию, хранящуюся в структурах, предоставляемых core.
## [client](client/client.md)
- Отрисовывает текущее состояние игры;
- Принимает ввод пользователя;
- Общается с сервером через network.
## [server](server/server.md)
- Общается с клиентом через network;
- Выполняет аутентификацию игроков;
- Содержит главный цикл обработки сообщений;
- Содержит игровой цикл, манипулирует состоянием игры через 3c;
- Делает запросы к базе данных через bd;
- Обрабатывает команды сервера.

# Модификация строений и синергий
Информация о строениях и их синергиях хранится в JSON-файлах, благодаря чему появляется возможность легко модифицировать сущестующие и добавлять новые строения и их синергии без перекомппиляции проекта. Базовая конфигурация зашита в код.
